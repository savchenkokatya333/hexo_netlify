发表于 2018-11-16 | 更新于 2019-01-18 | 分类于 数据结构

本文主要是参考网上，自己手撸了一遍代码，网上的代码看再多终究还是别人的，只有自己写写才能变成自己的东西，这里实现了：
通用工具方法：

  • isEmpty(self): 判断链表是否为空
  • size(self): 获得链表的长度
  • show(self, msg): 打印链表元素

插入数据：

  • add(self, value): 向链表尾部追加新元素
  • preAdd(s elf, value): 在链表头部插入新元素
  • insert(self, index, value): 在指定位置插入新元素

删除数据：

  • delByValue_1(self, value): 通过指定值来删除第一个满足要求的元素
  • delByValue_2(self, value): 同上
  • delByIndex(self, index): 通过指定位置删除元素

其他：

  • delDuplecate(self): 删除链表中重复的元素
  • reverse(self): 翻转链表

链表去重和翻转

这两个方法的思考容量对我现在来说有点大，弄了好久才写出来。
对于链表去重，单单用一个游标指针 cursor 好像不好做，于是又加了一个 pre 指针，具体思想用文字实在不知道怎么表达，直接理解代码吧，可以画图帮助理解。另外我是用集合 set 来装已经遍历且不重复的数据，为什么不用 list ，因为在 list 的存储结构是线性表，在线性表中找元素太慢，而 set 和 dict 的存储结构是 hash 表，找起元素来很快，这个在Two Sum中写过，当时的效率结果很明显。
对于翻转链表，开始的想法是遍历一遍，依次将元素加到 list 中，然后利用切片翻转 list ，最后遍历 list 并调用添加元素的方法重新建立链表，但这个方法好像有点违背我写这个的初衷了。于是思考其他方法，参看了下面的博客，一开始还看不懂，后面才理解了其思想。即用了一个临时变量 temp 来保存 cursor 的下个元素，致使 cursor 能不断向后遍历，具体还是看代码理解。
