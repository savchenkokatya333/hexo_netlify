【medium】Search in Rotated Sorted Array

发表于 2019-01-11 | 分类于 LeetCode日记

Suppose an array sorted in ascending order is rotated at some pivot unknown to you beforehand.

(i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]).

You are given a target value to search. If found in the array return its index, otherwise return -1.

You may assume no duplicate exists in the array.

Your algorithm’s runtime complexity must be in the order of O(log n).

思路分析：

  • 我先想的是遍历一遍列表，找出旋转点，令旋转点为center，这样center的左右两部分都是有序的，接下来就可以用二分查找做了，但题目限定了时间复杂度必须在O(log n)内，所以遍历一遍行不通，仔细分析列表的规律，可以发现：

    所以先进行二分查找的初始化，有了left、right、center后，判断当前的center满足上面哪一种情况，例如nums[center] > nums[left]，那么左边都是有序的，是从nums[left]到nums[center]递增的，继而可以通过左半部分的边界值判断target是否在左半部分，若不在，则令left = center+1，即进入右半部分查找。如此循环，直到left > right时break。

    Ps：其实就是改了二分查找的判断条件，标准的二分查找中，center左右都是有序的，比较target和center的大小，就可以知道该进入哪一半查找，而这里只有center的一边是有序的，所以判断条件变成了比较有序那一边的两个边界，之后就知道该进哪边查找了。具体见代码
