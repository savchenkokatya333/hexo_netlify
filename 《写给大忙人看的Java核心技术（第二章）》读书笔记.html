Java 中，所有参数（基本类型和对象引用）都是值传递

所不同的是，如果是基本类型，传递的就是其字面值的复制，如果是对象引用，传递的则是该对象在堆中的地址。

Java 到底是值传递还是引用传递？

final 和 static

首先说说final关键字，当它修饰于：

类：表示该类不能被继承了，一般好像不使用，限制性太大。

方法：那么该方法不能被子类重写。

变量：如果是基本类型，则在初始化后不能再修改了，如果是引用类型，即对象，那么该对象的地址不能更改了，但是对象的内容还是可以改的。

再看static关键字，当它修饰于：

方法：这种方法一般叫做静态方法了，它不再依赖于对象才能访问，只要类被加载，就可以通过类名来访问该方法。

变量：这种变量一般称为静态变量，它被该类的所有对象共享，即内存中只存在一个副本。

代码块：一般叫做静态代码块，通常用来做初始化，只在类加载的时候执行一次，优化性能。

5.实现一个描述平面上点的不可修改的Point类。提供一个设置具体点的构造函数、一个设置为原点的无参数构造函数，以及方法getX、getY、translate和scale。translate方法根据给定量在x和y方向上移动该点。scale方法根据给定因子在坐标系按比例变化。实现这些方法以便它们将新的Point对象作为结果返回。例如Point p = new Point(3, 4).translate(1, 3).scale(0.5)，应该设置p为坐标系上的点(2, 3.5)。

什么叫不可修改的类，就是该类的实例一旦创建，就不能再修改了，只能在创建时指定其内部的参数值，在官方文档中，也有创建不可变对象的指南，我也贴出来。

  1. Don’t provide “setter” methods — methods that modify fields or objects referred to by fields.
  2. Make all fields final and private.
  3. Don’t allow subclasses to override methods. The simplest way to do this is to declare the class as final. A more sophisticated approach is to make the constructor private and construct instances in factory methods.
  4. If the instance fields include references to mutable objects, don’t allow those objects to be changed:
    • Don’t provide methods that modify the mutable objects.
    • Don’t share references to the mutable objects. Never store references to external, mutable objects passed to the constructor; if necessary, create copies, and store references to the copies. Similarly, create copies of your internal mutable objects when necessary to avoid returning the originals in your methods.
